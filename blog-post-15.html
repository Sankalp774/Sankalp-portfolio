RefactorAI Gen 2 — The Project I Am Most Proud Of

RefactorAI Gen 2, latest push June 6, 2026, is the repository that makes my timeline feel coherent. It started in March 2026 as a full-stack upgrade and grew into a production-minded system: FastAPI backend, React 19 frontend, Monaco diff editor, LiteLLM integration, Docker, and Qwen 2.5-Coder driving refactoring workflows with static analysis guardrails. This was not a weekend demo. It was weeks of wiring services, designing API contracts, and making the UX feel fast even when models were slow. I cared about the boring parts — error envelopes, container builds, evaluation scripts — because boring parts are what separate prototypes from products. RefactorAI is also the story of iteration: the January code-analysis assistant experimented; Gen 2 commits. When I scroll through commits, I see my learning compressed — prompts tightened, security checks added, UI flows simplified. If my portfolio is a season, this repo is the podium finish. Not because it is flawless, but because it is the clearest proof of who I am becoming: a software fresher who builds AI systems with engineering discipline. I want this project to outlive the hype cycle around AI coding tools. Gen 2 is the project I send when someone asks what I can build end to end. Frontend, backend, models, containers, evaluation — one repo, one story, one standard I intend to keep raising. Watching a diff render in Monaco while a model proposes refactors still feels like magic — but magic I engineered, not wished for.
